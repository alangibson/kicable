# §6.3 — Component Library (G1)

> Corresponds to PRD §6.3 Component Library (FR-CL-01 – FR-CL-20).
> FR-CL-05 (shared team library) is G4 only. FR-CL-14 (server thumbnail generation) is G3+.
> FR-CL-21 (soft-delete scheduled cleanup) is G3+.

## Core Library

- [x] Ship built-in library with common connectors (Deutsch DT, AMP Superseal, TE MCP) and wire gauges (AWG and metric) (FR-CL-01)
- [x] Custom connector creation: part number, manufacturer, pin count, pin layout, gender, description (FR-CL-02)
- [x] Per-pin function labels (e.g., SIGNAL, GND, PWR) (FR-CL-03)
- [x] Export library to JSON; import library from JSON (FR-CL-04)
- [x] Version library items; schematic records which version of each component was used (FR-CL-06)

## Component Images (§6.3.1)

- [x] Attach images to components (JPEG, PNG, WebP, SVG; max 20 MB each) (FR-CL-07)
- [x] Categorize images by view type: `front`, `rear`, `side`, `assembled`, `installed`, `datasheet_scan`, `other`; multiple per category (FR-CL-08)
- [x] Designate one image as **primary**; display as thumbnail in library panel and canvas node (FR-CL-09)
- [x] Image gallery panel: drag-to-reorder, rename, recategorize, delete (FR-CL-10)
- [x] Store images in IndexedDB as ArrayBuffers keyed by `component_id/image_id` (FR-CL-11, FR-CL-13)
- [x] Embed images as Base64 data URIs in JSON library exports; extract on import (FR-CL-12)

## Component STEP Files (§6.3.2)

- [x] Attach one STEP file per component (`.step`/`.stp`; max 200 MB; warn at 50 MB) (FR-CL-15, FR-CL-16)
- [x] Display STEP file metadata: filename, size, upload date (FR-CL-17)
- [x] Download button for STEP file; correct MIME type (`model/step`) and `Content-Disposition: attachment` (FR-CL-18)
- [x] No in-browser STEP rendering; link to external viewer is acceptable (FR-CL-19)
- [x] Include STEP file in `.chd` zip as `step/<component_id>.step` (FR-CL-20)
