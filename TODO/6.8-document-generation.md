# §6.8 — Document Generation (G1)

> Corresponds to PRD §6.8 Document Generation (FR-DG-01 – FR-DG-08).
> All generation runs in-browser via PDFKit (browser build) and ExcelJS; output via FileSaver.js.

- [ ] **Wire List:** table of all wires — Wire ID, From Connector/Pin, To Connector/Pin, Gauge, Color, Overall Length, Signal Name; export PDF, XLSX, CSV (FR-DG-01)
- [ ] **BOM:** aggregated connectors, terminals, seals, wire by part number with quantities + dedicated Protective Materials section; export PDF, CSV (FR-DG-02)
- [ ] **Cut Sheet:** per-wire/cable manufacturing sheet including:
  - Wire ID, signal name, gauge, color, overall length
  - Segment lengths table (if segments defined)
  - End A and End B: connector, pin, strip length, strip type, tinning, terminal, insertion depth, notes
  - Cable-level: jacket strip lengths, shield treatment, drain/pigtail lengths per end
  - Cable inner conductors: sub-table with per-conductor strip definitions
  - Protection table: all coverage spans in offset order with material, extents, layer, calculated quantity
  - Export PDF, CSV (FR-DG-03)
- [ ] **Connector Face Diagram:** auto-rendered pin-face view per connector with pin assignments; inset primary image if available; export PDF (FR-DG-04)
- [ ] **Assembly Drawing:** schematic-level PDF with protective material coverage bands as colored overlays, material legend, split/join callouts, fan-out length annotations (FR-DG-05)
- [ ] All documents export to PDF and CSV; Wire List additionally exports to XLSX (FR-DG-06)
- [ ] Customizable templates: company logo, header/footer fields, color scheme (FR-DG-07)
- [ ] Run DRC validation before generation; highlight errors in UI; block export on errors, allow on warnings/info (FR-DG-08)
- [ ] Document generation for 200-wire harness completes in < 3 s in-browser (success metric)
